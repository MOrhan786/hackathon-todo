### 11. Responsive Breakpoints

#### Mobile (320px - 474px)
- Single column task grid
- Bottom navigation visible
- Sidebar collapsed behind hamburger menu
- Form fields stacked vertically
- Compact spacing (16px base)

#### Mobile Landscape (475px - 639px)
- Single column task grid
- Bottom navigation visible
- Slightly wider content area
- Improved spacing

#### Tablet Portrait (640px - 767px)
- Two column task grid
- Bottom navigation visible
- Sidebar collapsed behind hamburger menu
- Better form layout

#### Tablet Landscape (768px - 1023px)
- Two column task grid
- Bottom navigation hidden
- Sidebar expanded by default
- Optimized form layout

#### Desktop (1024px+)
- Three column task grid (expands to four on larger screens)
- Sidebar always visible
- Full header navigation
- Optimal form layout

### 12. Animation Specifications

#### Card Entry Animation
- Initial: translateY(20px) opacity(0)
- Final: translateY(0) opacity(1)
- Duration: 300ms
- Easing: cubic-bezier(0.4, 0, 0.2, 1)

#### Button Hover
- Scale: 1.05
- Transition: 200ms ease-in-out
- Shadow: Increase shadow intensity

#### Loading Spinner
- Rotation: Continuous 360Â°
- Duration: 1000ms
- Easing: Linear

#### Form Field Focus
- Border color: Primary gradient
- Box shadow: Primary glow effect
- Transition: 200ms ease

### 13. Accessibility Specifications

#### Focus States
- Outline: 2px solid hsl(175, 80%, 50%) (Primary)
- Outline-offset: 2px
- Visible on all interactive elements

#### Screen Reader Labels
- ARIA labels for all form elements
- Descriptive alt text for icons
- Landmark roles for navigation regions

#### Color Contrast
- Text/Background: Minimum 4.5:1 ratio
- Interactive elements: Enhanced contrast
- Focus indicators: High visibility

### 14. Component Specifications

#### Buttons
- **Primary**: Gradient background, white text, rounded-md
- **Secondary**: Transparent with border, colored text
- **Destructive**: Red background, white text
- **Ghost**: Transparent, colored text only
- **Sizes**: Small (32px), Medium (40px), Large (48px)

#### Inputs
- **Text**: Rounded corners, subtle border, proper padding
- **Textarea**: Similar to text input but taller with min-height
- **Select**: Custom dropdown with chevron indicator
- **Validation**: Error state with red border and message

#### Cards
- **Background**: hsl(220, 25%, 12%) (Card)
- **Border**: 1px solid hsl(220, 20%, 25%) (Input)
- **Padding**: 24px internal spacing
- **Shadow**: 0 4px 6px rgba(0, 0, 0, 0.3)

#### Modals
- **Backdrop**: Semi-transparent overlay
- **Content**: Centered card with proper spacing
- **Close**: X button in top-right corner
- **Animation**: Fade-in with slide-up effect

This UI specification provides detailed visual mockups and interaction patterns for implementing the Todo List application with the Dark Luxury Design System and Vibrant Gradients aesthetic.
