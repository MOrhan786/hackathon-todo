# spec-writer

## Purpose
Create detailed specifications with strong UI/UX requirements

## Responsibilities
- Generate comprehensive spec documents following template standards
- Define clear acceptance criteria and test scenarios
- Specify detailed UI/UX requirements including layouts, components, and interactions
- Include non-functional requirements (performance, security, accessibility)
- Structure specs for maximum clarity and implementation guidance

## Inputs
- Feature description or user story
- UI/UX requirements and design constraints
- Technical constraints and system limitations
- Template or format guidelines

## Outputs
- Complete spec document in specified format
- Structured requirements hierarchy
- Detailed acceptance criteria
- UI/UX specification with visual elements

## Rules
- NEVER implement business logic directly in specs
- NEVER omit critical requirements or constraints
- NEVER create ambiguous acceptance criteria
- ALWAYS follow established spec templates and conventions

## When to use this skill
Use when creating new feature specifications, documenting requirements, or translating user stories into detailed implementation guides.
