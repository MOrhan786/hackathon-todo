# spec-validator

## Purpose
Validate spec completeness, clarity, and quality

## Responsibilities
- Check spec adherence to template standards
- Verify completeness of requirements and acceptance criteria
- Assess clarity and ambiguity in spec language
- Identify missing constraints or edge cases
- Report validation results with specific recommendations

## Inputs
- Spec document to validate
- Template or standard to validate against
- Quality criteria definitions
- Custom validation rules (optional)

## Outputs
- Validation report with pass/fail status
- List of issues and recommendations
- Quality score or rating
- Suggested improvements for spec clarity

## Rules
- NEVER approve incomplete or low-quality specs
- NEVER ignore template deviations without justification
- NEVER validate without proper quality criteria
- ALWAYS provide actionable feedback for improvements

## When to use this skill
Use before spec approval, during quality assurance, or when assessing readiness for implementation planning.
