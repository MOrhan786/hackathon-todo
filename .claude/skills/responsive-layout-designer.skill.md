# responsive-layout-designer

## Purpose
Design mobile-first responsive layouts across all breakpoints

## Responsibilities
- Create mobile-first layout structures that scale appropriately
- Define responsive breakpoints for different device sizes
- Ensure content hierarchy remains clear across devices
- Optimize component arrangements for each breakpoint
- Maintain usability and accessibility across all views

## Inputs
- Content structure and information hierarchy
- Target device specifications and screen sizes
- Design system constraints and components
- Usability requirements and user flow considerations

## Outputs
- Responsive layout specifications with breakpoints
- Component arrangement guidelines for each size
- Grid system configurations
- Mobile adaptation strategies

## Rules
- NEVER prioritize desktop layouts over mobile
- NEVER ignore critical content at smaller sizes
- NEVER sacrifice usability for visual appeal
- ALWAYS maintain consistent navigation patterns

## When to use this skill
Use when designing interfaces that must work across multiple devices, creating adaptive layouts, or optimizing user experiences for different screen sizes.
